# CLAUDE.md

## Plan & Review

### Before starting work
- Always enter plan mode to make a plan
- After getting the plan, write it to `.claude/tasks/TASK_NAME.md`
- The plan should be a detailed implementation plan with reasoning, and tasks broken down into steps
- If the task requires external knowledge or a certain package, research it first (use Task tool for research)
- Don't over plan it, always think MVP
- Once you write the plan, ask for review. Do not continue until the plan is approved

### While implementing
- Update the plan as you work
- After completing tasks in the plan, append detailed descriptions of the changes made so the work can be easily handed off to other engineers
